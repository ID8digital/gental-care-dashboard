Deployment guide — step by step
1
Deploy the server to Render (free tier works for pilot)
Push the gental-care-orm/ folder to a GitHub repo. Go to render.com → New Web Service → connect your repo. Render auto-detects the render.yaml config. Set region to Singapore (closest to Kenya).
# Your live webhook URL will be:
https://gental-care-orm-engine.onrender.com/webhook/meta
https://gental-care-orm-engine.onrender.com/webhook/tiktok
https://gental-care-orm-engine.onrender.com/health
2
Set environment variables in Render dashboard
In Render → your service → Environment. Add all values from .env.example. Start with just ANTHROPIC_API_KEY and META_* — Zoho and TikTok can come later.
Required for MVP:
ANTHROPIC_API_KEY=sk-ant-...
META_APP_SECRET=from Meta Developer portal
META_PAGE_ACCESS_TOKEN=long-lived page token
META_VERIFY_TOKEN=gental_care_webhook_2025
3
Create a Meta Developer App and connect the Gental Care page
Go to developers.facebook.com → Create App → Business type. Add "Webhooks" product. In Webhooks → Page → Subscribe to: messages, feed, mention. For Instagram: add "Instagram Graph API" → subscribe to comments, messages. Enter your Render webhook URL and verify token.
4
Generate a long-lived Page Access Token
In Meta → Graph API Explorer → select your Page → get token → extend it via the token debugger (60-day tokens expire — automate renewal or use a System User token which never expires). This token lets the engine post replies and hide comments on behalf of the page.
5
Connect Zoho Desk for escalation tickets
Zoho Desk → Settings → Developer Space → Self Client → Generate OAuth token with scope Desk.tickets.CREATE, Desk.tickets.READ. Add ZOHO_OAUTH_TOKEN, ZOHO_ORG_ID, ZOHO_DEPARTMENT_ID to Render. Tickets auto-create with tag, priority, and full context.
6
TikTok (add after Meta pilot is stable)
TikTok for Developers → create app → request Comment API scope. TikTok requires app review (1–3 weeks). Once approved, add TIKTOK_ACCESS_TOKEN to Render. Note: TikTok's API only supports comments on your own videos — no DMs via API currently.
7
Test end-to-end before going live
Comment on a Gental Care post from a test account. Check Render logs to see the webhook fire, Claude classify it, and the reply post back. Check /health endpoint shows all integrations as connected. Run the local test suite first: cd server && node test-local.js
⚠ Meta app review note
For posting replies to comments and DMs on behalf of a Page, Meta requires the pages_manage_engagement and pages_messaging permissions. These go through Meta's App Review process (typically 1–5 business days). In the meantime, the engine can receive and classify webhooks — it just can't post replies until approved. You can test everything in Development Mode using only the Page admin's account.
